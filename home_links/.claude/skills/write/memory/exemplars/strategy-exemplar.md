## Key Patterns

### Emphasis
- `*italics*` for key concepts: *programmable*, *a few will program for the many*
- `**bold**` for inline emphasis: "for both users **and** the company"
- `"**Named Concept**"` for technical systems
- No triple emphasis (different from two-pagers)

### Structure
- Simple cover page with just "Pre-read" or document type
- Subtitle with version/audience + date on separate line
- Blue text for version identifier
- Sections follow: Context → Vision → Tech → Execution
- "From X to Y" pattern in section titles

### Content Patterns
- "Where most platforms... we're building toward..." - differentiation
- "If we get this right, Spotify will become..." - conditional future
- "*a few will program for the many*" - aphoristic principles (italicized)
- Inline links to related docs for context
- Dense paragraphs (appropriate for pre-reads)
- Bulleted lists for concrete user capabilities

### Linking Style
- `[descriptive text](url)` inline with flow
- "(see our 2-min video overview [here](url) **WIP**)" for supplementary content
- "[this tab]()" for internal navigation

### Context Opening

```
We are in a very different place than we were at the last AI Strategy Day. In March, our [update](url) included a broad survey of AI activity across the industry, followed by updates across our consumer experience: what was launching, what we were experimenting with, and what we hoped to figure out.
```

### Vision Statement
```
We envision a *programmable* Spotify, where in addition to users receiving a world class recommendation experience (which is getting even better due to our GenRecs investments), they can also steer it, shape it, and co-create it.
```

### Principle Pattern
```
Key to this vision is a foundational principle: *a few will program for the many*. Many people will make small efforts to control and personalize for themselves, but we expect that a small percentage of power users will craft the most complex and expressive logic, creating reusable prompts that generate value at scale across the user base.
```

### Future Capabilities List
```
In our future as a programmable media platform, users will
• Write, remix and share prompts to generate music playlists, talk sessions, and routines.
• Bring their own identity to Spotify through a user system prompt that guides AI behavior across their experience.
• Talk to Spotify anytime via chat or voice, and have a memoryful, evolving relationship with the Agent.
```
